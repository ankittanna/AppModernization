Description

DeleteCtrl

Controller of the appModernizationApp This controller handles the view model of the delete/cancel reservation. This is responsible for cancelling the existing reservations.

It initially fetches the details of the reservation by requesting the data based on reservationId. The reservationId is then used to cancel the reservation. This fetching is done using HRS.getRegisteredData(reservationID) which fetches the data and stores the data in reservedData and manipulates the data using UtilitiesService to make it usable for the View-Model binding. Finally, when all the manipulations is done the reservation details are stored in $scope.reservationDetails

Methods

  • deleteReservation()

    This method calls the service HRS.cancelReservation(reservationId) for cancelling the reservation. It picks the reservation id on the scope variable and calls the service for cancelling the reservation. Once the cancel reservation service is called successfully it then moves back to search reservations page.
